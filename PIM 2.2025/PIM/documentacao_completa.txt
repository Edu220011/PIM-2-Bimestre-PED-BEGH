SUMÁRIO EXECUTIVO
=================

Este documento apresenta uma documentação técnica completa do Sistema Acadêmico PIM 3.0, 
desenvolvido em Python com interface gráfica CustomTkinter. O sistema implementa um modelo 
cliente-servidor para gerenciamento acadêmico, incluindo autenticação, CRUD de usuários, 
turmas, matérias, notas e atividades.

1. INTRODUÇÃO
==============

1.1 OBJETIVO
------------
Este documento visa fornecer uma documentação técnica detalhada de todos os arquivos do 
sistema, separando-os por componentes Server e Client, incluindo análise de funções, 
indentação, formato de código e lógica implementada.

1.2 ESCOPO
----------
- Arquivos do lado Server (proxy e dados)
- Arquivos do lado Client (interface e utilitários)
- Análise técnica completa de cada arquivo
- Estrutura seguindo normas ABNT

1.3 METODOLOGIA
---------------
- Análise estática do código fonte
- Documentação de funções e métodos
- Verificação de padrões de codificação
- Estruturação hierárquica por componentes

2. ARQUITETURA GERAL DO SISTEMA
===============================

2.1 COMPONENTES PRINCIPAIS
--------------------------
- **Server**: Proxy TCP para sincronização de dados
- **Client**: Interface gráfica para usuários finais
- **Dados**: Arquivos JSON para persistência

2.2 TECNOLOGIAS UTILIZADAS
--------------------------
- Linguagem: Python 3.x
- Interface: CustomTkinter
- Criptografia: bcrypt
- Comunicação: Socket TCP
- Persistência: JSON

3.1 COMPONENTE SERVER
=====================

3.1.1 ARQUIVO: server.py
------------------------

**Localização**: Server\conexao\maq1\server.py
**Tipo**: Script Python
**Função Principal**: Servidor proxy TCP para sincronização de dados

**DESCRIÇÃO GERAL**:
Implementa um servidor TCP que escuta conexões na porta 65432, processando 
requisições de carregamento e salvamento de dados JSON entre clientes.

**FUNÇÕES PRINCIPAIS**:

def __init__(self):
    """Inicializa o servidor com configurações básicas"""
    - Define host e porta
    - Cria socket TCP
    - Inicializa dicionário de dados

def start(self):
    """Inicia o servidor e loop de escuta"""
    - Liga o servidor
    - Aceita conexões continuamente
    - Processa requisições em threads

def handle_client(self, client_socket):
    """Processa requisição de um cliente"""
    - Recebe comando (carregar/salvar)
    - Executa operação correspondente
    - Retorna resultado ou confirmação

def carregar_dados_do_servidor(self, filename):
    """Retorna dados JSON solicitados"""
    - Carrega arquivo do disco
    - Trata erros de decodificação
    - Retorna dados ou lista vazia

def salvar_dados_no_servidor(self, filename, data):
    """Salva dados JSON no servidor"""
    - Salva dados no arquivo correspondente
    - Trata erros de escrita
    - Confirma operação

**INDENTAÇÃO E FORMATO**:
- Indentação: 4 espaços (PEP 8)
- Comprimento máximo de linha: 88 caracteres
- Comentários: Docstrings em português
- Estrutura: Classe Server com métodos

**LÓGICA IMPLEMENTADA**:
O servidor utiliza um modelo de comunicação síncrona baseada em comandos de texto. 
Cada cliente envia um comando seguido do nome do arquivo, e o servidor responde 
com os dados solicitados ou confirmação de salvamento.

    """Estabelece conexão TCP com servidor"""
    - Cria socket cliente
    - Conecta na porta 65432
    - Retorna socket conectado

def carregar_dados_do_servidor(filename):
    """Solicita dados do servidor"""
    - Conecta ao servidor
    - Envia comando de carregamento
    - Recebe e retorna dados

def salvar_dados_no_servidor(filename, data):
    """Envia dados para salvamento"""
    - Conecta ao servidor
    - Envia comando de salvamento
    - Confirma operação

**MÉTODOS PRINCIPAIS**:

    """Inicializa aplicação GUI"""

    - Define geometria da janela
    - Inicializa variáveis de estado

def _build_login(self):

    - Validação de entrada
    - Botão de autenticação

def autenticar(self):
    """Processa login do usuário"""
    - Valida credenciais
    - Redireciona para dashboard apropriado

def _build_dashboard(self):
    """Cria interface principal baseada no tipo de usuário"""

**LÓGICA**: Interface baseada em estados, com diferentes painéis por tipo de usuário.
